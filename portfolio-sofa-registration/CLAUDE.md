# O mně jako UX designerovi

## Přístup k designu

### Milý
- **Error messages s empatií**: "Zkontrolujte prosím @ a koncovku" místo "Chyba!"
- **User-first thinking**: Respekt k času a pozornosti uživatele
- **Přátelský, ale ne infantilní**: Profesionální bez corporate bullshitu

### Chytrý
- **Data-driven rozhodnutí**: 78% redukce kroků, 55% redukce polí - měřitelné výsledky
- **Moderní UX patterns**: Show/hide password místo duplikátů, auto-advance SMS inputs
- **Vím kdy porušit pravidla**: Challenge legacy assumptions (duplicitní pole, email aktivace)

### Zkušený
- **Reduction over addition**: Nejlepší UX je ten, na který si uživatel nevzpomene
- **Respekt k user contextu**: Když user právě zadal credentials, nedávám mu je zadávat znovu
- **Progressive disclosure**: Power features pro power users, simple flow pro simple users

## Filozofie: Zjednodušování

**Klíčová otázka není "co přidat?" ale "co odstranit?"**

- Každé pole je příležitost k opuštění
- Každý krok je friction point
- Každá duplicita je otrava

**Výsledek:**
- 8-9 kroků → 2 kroky
- 11+ polí → 5 polí
- 100% eliminace duplicit
- Time to value: jednotky sekund místo minut

## Když navrhovat onboarding

**Odpověď: Skoro nikdy.**

Pokud uživatelé potřebují tutoriál aby pochopili vaši appku, problém není chybějící onboarding - problém je vaše appka.

**Místo toho:**
- Self-evident design
- Clear empty states s CTA
- Contextual help when needed
- Learn by doing, not by watching

---

*Tento projekt demonstruje, že dobrý UX není o přidávání features - je o eliminaci všeho mezi user intent a user value.*
